# CIC-IDS2017 Split Protocol

> Version: 1.0
> Frozen: 2026-03-23
> Applies to: CIC-IDS2017 only (will be extended uniformly to CSE-CIC-IDS2018, UNSW-NB15 in later phases)

---

## 1. Scope and Input

This document defines the immutable train/validation/test split protocol for all
Phase 2 (and later) experiments on the CIC-IDS2017 data asset.

**Formal input:** The master feature table frozen in Phase 1:
- `D:/ids_project_data/processed/cicids2017/cicids2017_flows_master.parquet`
  (1,901,644 rows; Group A flows only: matched_flag=True AND binary_label NOT NULL)

This protocol does **not** refer back to raw PCAP or to the public CIC label CSVs
as direct training input. The Phase 1 frozen feature tables are the authoritative
input to all Phase 2 experiments.

This protocol covers **only** the flows present in the master table. Ambiguous,
unmatched, unmapped, and parse-fail flows (Groups B–E) are already excluded from
master and are not part of any split.

---

## 2. Split Unit

**Day-level split.** Each capture day is assigned as a whole to exactly one split.

Per-flow random assignment is **not permitted** for the following reasons:

- Flows from the same session or time window share IP pairs, timing, and
  statistical properties. Mixing them across splits inflates performance estimates
  and does not reflect real deployment generalization.
- Day-level splits are consistent with leakage-aware design (Decision D5).
- Day-level splits make the split reproducible without a random seed and make
  day-specific distribution shifts visible in evaluation.

---

## 3. Fixed Day → Split Mapping

The following assignment is frozen and immutable:

| Day       | Split | Rationale |
|-----------|-------|-----------|
| Monday    | train | Benign-only day; establishes normal traffic baseline |
| Tuesday   | train | Brute-force attacks (FTP-Patator, SSH-Patator) |
| Wednesday | train | DoS/DDoS attacks (DoS Hulk dominant); largest attack volume |
| Thursday  | val   | Web attacks + Infiltration; held out for model selection |
| Friday    | test  | Reconnaissance + Botnet; held out for final evaluation |

**This mapping must not be changed after freeze.** See Section 7 for invalidation
conditions.

---

## 4. Split Assignment Artifact

The split assignment is stored as a **sidecar CSV**, separate from the master
parquet. The master table must not be modified.

Path: `D:/ids_project_data/processed/cicids2017/cicids2017_split_assignment.csv`

Required columns:
- `global_flow_id` — matches master table; every master row has exactly one entry
- `day` — the capture day (redundant copy for auditability)
- `split` — one of `train`, `val`, `test`

Generated by: `scripts/freeze_phase2_cicids2017.py`

## 5. Usage Constraints for All Downstream Experiments

The following rules apply to every experiment that consumes this split:

### 5.1 Scaler / Normalizer / Imputer Fitting
Any preprocessing object (StandardScaler, MinMaxScaler, imputer, PCA, etc.) must
be **fitted on the train split only** and then applied (transform only) to val and
test. Fitting on val or test data constitutes leakage.

### 5.2 Threshold Selection
Decision thresholds may be selected using the **val split** (e.g., to meet a
fixed-FPR budget). The test split must not be used for threshold selection.

### 5.3 Calibration
Score calibration (Platt scaling, isotonic regression, etc.) must be fitted on the
**train split** (or a held-out portion of it). Val and test must not be used for
calibration fitting.

### 5.4 No Post-Hoc Re-Assignment
After this protocol is frozen, the day→split mapping must not be modified to
improve model performance. Any change to the mapping requires a full protocol
re-freeze under a new version number and invalidates all prior results.

### 5.5 Cross-Domain Experiments
For cross-domain transfer experiments (using CSE-CIC-IDS2018 or UNSW-NB15 as
target domain), target-domain data must not be used for model tuning, threshold
selection, or calibration fitting. This constraint is separately documented in
Decision D3 and applies irrespective of this split protocol.

## 6. Relationship to Master Table

| Item | Location | Modified by this protocol? |
|------|----------|---------------------------|
| Flow features | `*_features.parquet`, master parquet | **No** |
| Flow labels | `binary_label`, `coarse_family` columns in master | **No** |
| Split assignment | `cicids2017_split_assignment.csv` (sidecar) | Yes — this is the only artifact written |
| Per-day logs | `D:/ids_project_data/logs/cicids2017/{day}/` | **No** |

The master parquet is read-only from this point forward. Any update to master
requires Phase 1 re-execution and Phase 2 re-freeze.

---

## 7. Invalidation Conditions

This protocol freeze is invalid and must be redone if any of the following
changes:

1. `cicids2017_flows_master.parquet` is regenerated or modified
2. Any of the five `{day}_features.parquet` files are regenerated or modified
3. `data/external/label_mapping_master.csv` is modified
4. `docs/flow_schema.md` is modified (changes canonical feature definitions)
5. `docs/exclusion_policy.md` is modified (changes which flows are in master)
6. The day→split mapping in Section 3 of this document is revised

When re-freezing, increment the version number, update the `Frozen:` date, and
re-run `scripts/freeze_phase2_cicids2017.py --force`. All downstream model
results trained on the old split are invalidated.

---

## 8. What Has NOT Been Done

This document governs data governance only. As of Phase 2 freeze:

- No Suricata scanning has been performed
- No LR / RF / XGBoost model has been trained on the full master table
- No 1D-CNN / LSTM model has been trained
- No threshold analysis or calibration has been performed on full-day data
- No cross-domain transfer experiments have been run
- No CSE-CIC-IDS2018 or UNSW-NB15 processing has begun
